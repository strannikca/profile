Mobile development

Mobile platform were far away from my development scope for quite a while. In particular, until Windows Phone 7.5 release. Even more – until first Nokia smartphone with this OS – Lumia 800 – finally reached Ukraine. It was 2012. I was almost completely satisfied with this mobile OS and still like it much more than Android and iOS, and use on my primary phone. Unfortunately, in Canada most of developers and service companies simply ignore this interesting operating system, so I have to possess Android smartphone as secondary device.

In the same 2012 I started to learn the application development process for this platform. It didn’t seem very hard as looked the same like developing desktop applications with WPF, just with mobile UI. However, it appeared (as a surprise to me) that Silverlight, which was the only available technology for Windows Phone development, was an extremely light version of .NET, without some crucial built-in features. And the most challenges were to overcome this restriction. Not that many open source libraries were available that days, so I even had to implement PNG encoding by myself. Well, it was interesting to finally sort out with compression algorithms as well as CRC and PNG structure itself. Definitely, mobile development was not my primary occupation. I made two application for internal use in the company I worked for and only two apps published to the store under my name.

As you may know from profile page, there was a third project. It was the manager of personal finances. Yet again, I needed it for myself as no existing manager implements all the features I need and/or not the convenient way. Some of them are overloaded with features and too complex for everyday use, others easy and fast to use, but too simple. My core idea was in building modularized app, where a user could turn on and off the modules. The activation of module should not only add features on separate pages, but change one or all of: menu items (supposed to be partly customizable), interface and functionality of core pages as well as dependent modules’ appearance. So, the general idea was in finding balance among functionality, customization and simplicity of use. However, there were reasons to switch to game development exclusively. Initially, I planned to do both projects, but it was too much for a single person.
